## Local

### Label Studio

Link: [Label Studio](https://labelstud.io/)

Install:
```shell
$ python3 -m venv $HOME/.venv
$ source $HOME/.venv/bin/activate
$ pip install label-studio
$ pip install label-studio-converter
$ rehash
```

Run:
```shell
$ cd label-studio
$ source .venv/bin/activate
$ label-studio --data-dir $HOME/data/cv/tools/label-studio
```

Run with local storage config:
```
export LABEL_STUDIO_LOCAL_FILES_SERVING_ENABLED=true
export LABEL_STUDIO_LOCAL_FILES_DOCUMENT_ROOT=<absolute-path-to-datasets-dir>
label-studio --data-dir $HOME/data/cv/tools/label-studio
```
Note: Local storage is a dir with a bunch of datasets available for importing into `Label Studio`. Currently YOLO and COCO datasets are supporting.
### Import COCO/YOLO dataset

Link: [Importing Local YOLO Pre-Annotated Images](https://labelstud.io/blog/tutorial-importing-local-yolo-pre-annotated-images-to-label-studio/)

* Start `Label Studio` with local storage
* Create a new project 
* Head to the project settings and select `Cloud Storage`
* Click `Add Source Storage` and select `Local files`
* Set the `Absolute local path` to dataset full path (if local storage root is `/datasets` then this path should contain `/datasets/dataset1`)
* Select `Add storage` to confirm the setup
* Check image access using URL `http://localhost:8080/data/local-files/?d=dataset1/<file-name>.jpg`
* Convert annotations:
```shell
$ label-studio-converter import coco \
-i <path/to/coco.json> \
-o <path/to/label-studio.json> \
--image-root-url "/data/local-files/?d=<relative/path/to/dir>"
```
Note: `--image-root-url` specify the root URL which contains relative path where to find dataset file (e.g. `cars/images`). 

* Return to `Label Studio` project
* Click `Settings` and `Labelling Interface` then `Code`
* Paste the content of `label-studio.label_config.xml` file and click `Save`
* Click `Import` and select `label-studio.json` file
