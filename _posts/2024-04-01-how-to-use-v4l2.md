# Install

```shell
# Ubuntu
$ sudo apt install v4l-utils
# ArchLinux
$ sudo pacman -S v4l-utils
```
Includes:
* `v4l2-ctl` - controls V4L2 devices from the command line, allows query device capabilities, set device parameters and capture video and still images
# Tools

List all available V4L2 devices:
```shell
$ v4l2-ctl --list-devices
```

List all available controls for the device:
```shell
$ v4l2-ctl -d 0 -l
User Controls
...
Camera Controls
...
```

Set the value of a control for `/dev/video0` device:
```shell
$ v4l2-ctl -d 0 -c <control>=<value>
```

Display the current video format of `/dev/video0` device:
```shell
$ v4l2-ctl -d 0 -V
Format Video Capture:  
       Width/Height      : 800/600  
       Pixel Format      : 'MJPG' (Motion-JPEG)  
       Field             : None
...
```

List all supported formats by `/dev/video0` device:
```shell
$ v4l2-ctl -d 0 --list-formats
ioctl: VIDIOC_ENUM_FMT
        Type: Video Capture

        [0]: 'YUYV' (YUYV 4:2:2)
        [1]: 'MJPG' (Motion-JPEG, compressed)
```

List supported formats (extended) by `/dev/video0` device:
```shell
$ v4l2-ctl -d 0 --list-formats-ext
ioctl: VIDIOC_ENUM_FMT
        Type: Video Capture

        [0]: 'YUYV' (YUYV 4:2:2)
                Size: Discrete 640x480
...
```

# Using

## Using with gstreamer

* Show video stream in a window:
```shell
$ gst-launch-1.0 \
 v4l2src device=/dev/video0 ! \
 videoconvert ! \
 autovideosink
```

* Save several frames in separate files (RAW format) 
```shell
$ GST_DEBUG=WARN gst-launch-1.0 -e \
 v4l2src device=/dev/video0 num-buffers=3 ! \
 videoconvert ! \
 videoscale ! "video/x-raw,width=640,height=480,format=RGB" ! \
 multifilesink location="frame-%05d.raw"
```

* Save several frames in separate files (JPEG format)
```shell
$ gst-launch-1.0 \
 v4l2src device=/dev/video0 num-buffers=3 ! \
 videoconvert ! \
 videoscale ! "video/x-raw,width=640,height=480,format=RGB" ! \
 jpegenc ! multifilesink location="frame-%05d.jpg"
```
# Misc

## Setup V4L2 loopback video devices

Streaming using gstreamer:
```shell
# Streaming (RGB, 1024x768)
$ gst-launch-1.0 videotestsrc pattern=smpte ! "video/x-raw,width=1024,height=768,format=RGB" ! v4l2sink device=<output-device>
# Streaming NV12, 1024x768
$ gst-launch-1.0 videotestsrc pattern=smpte ! "video/x-raw,width=1024,height=768,format=NV12" ! v4l2sink device=<capture-device>
# Capture (RGB, 1024x768)
$ gst-launch-1.0 -e v4l2src device=/dev/video94 ! "video/x-raw,format=RGB" ! videoconvert ! autovideosink sync=false
# Capture (NV12, 1024x768)
$ gst-launch-1.0 -e v4l2src device=/dev/video94 ! "video/x-raw,format=NV12" ! videoconvert ! autovideosink sync=false
```
Streaming using ffmpeg:
```shell
$ sudo apt install ffmpeg
# BGR, 1024x768
$ ffmpeg -re -f lavfi -i testsrc=duration=1000000:size=1280x768:rate=25 -pix_fmt bgr24 -f v4l2 <output-device>
# NV12, 1024x768
$ ffmpeg -re -f lavfi -i testsrc2=duration=1000000:size=1280x768:rate=25 -pix_fmt nv12 -f v4l2 <capture-device>
```

Streaming using ffmpeg from file:
```shell
$ ffmpeg -stream_loop -1 -re -i <video-file> -vf scale=1280:1024 -pix_fmt rgb24 -f v4l2 <output-device>
```

Display video stream:
```shell
$ ffplay <capture-device>
```
## v4l2loopback

Install:
```shell
$ sudo apt install v4l2loopback-dkms v4l2loopback-utils
```

Create two video devices:
```shell
sudo modprobe v4l2loopback \
devices=2 \
video_nr=94,95 \
exclusive_caps=1,1 \
card_label="CAM1","CAM2"
```

Check the presence of video devices:
```shell
$ ls -1 /sys/devices/virtual/video4linux
video94
video95
```

Destroy video devices:
```shell
# Stop all consumers of V4L2 devices
$ sudo rmmod v4l2loopback
```

## akvcam

### Build and install

Build and install driver:
```shell
$ git clone https://github.com/webcamoid/akvcam.git
$ cd akvcam/src
$ make
$ sudo make dkms_install
$ ls /lib/modules/$(uname -r)/updates/dkms/akvcam*
/lib/modules/6.11.0-29-generic/updates/dkms/akvcam.ko.zst
```

Uninstall driver:
```shell
$ cd akvcam/src
$ sudo make dkms_uninstall
```

Manual driver installation (if only packaging was used):
```shell
$ sudo dkms install akvcam/${VERSION}
```
### Configure

```shell
$ sudo mkdir -p  /etc/akvcam
$ sudo tee /etc/akvcam/config.ini > /dev/null <<EOF
[Cameras]
cameras/size = 2

cameras/1/type = output
cameras/1/mode = mmap, userptr, rw
cameras/1/description = VCAM1 (output device)
cameras/1/formats = 1
cameras/1/videonr = 93

cameras/2/type = capture
cameras/2/mode = mmap, rw
cameras/2/description = VCAM
cameras/2/formats = 2
cameras/2/videonr = 94

[Formats]
formats/size = 2

formats/1/format = RGB24
formats/1/width = 1280
formats/1/height = 1024
formats/1/fps = 30

formats/2/format = RGB24, YUY2
formats/2/width = 1280
formats/2/height = 1024
formats/2/fps = 30

[Connections]
connections/size = 1

connections/1/connection = 1:2
EOF
```

### Usage

Load kernel module:
```shell
$ sudo modprobe akvcam
```

Unload kernel module:
```shell
$ sudo rmmod akvcam
```

Load kernel module on boot:
```shell
echo akvcam | sudo tee /etc/modules-load.d/akvcam.conf
```

Checking compliance:
```shell
$ v4l2-compliance -d /dev/video94 -f -s
...
Total for akvcam device /dev/video93: 56, Succeeded: 56, Failed: 0, Warnings: 0
$ v4l2-compliance -d /dev/video95 -f -s
...
Total for akvcam device /dev/video94: 57, Succeeded: 57, Failed: 0, Warnings: 0
```

# Development

The `libv4l2` small library provides handlers to call V4L2 open/ioctl/close/poll methods. Instead of just providing the raw output of the device, it enhances the calls in the sense that it will use `libv4lconvert` library to provide more video formats (`RGB` and `BGR`) and to enhance the image quality.

Adding to the project:
```cmake
find_package(PkgConfig)  
  
pkg_check_modules(LibV4l2 REQUIRED IMPORTED_TARGET libv4l2)
```

Link with target:
```cmake
target_link_libraries(${TARGET} PUBLIC PkgConfig::LibV4l2)
```

## Capturing

Capturing video stream using V4L2:
* Save source code of [capture.c](https://www.kernel.org/doc/html/v4.9/media/uapi/v4l/capture.c.html)
* Compile source code: `$ gcc capture.c -o capture`
* Run capturing:
```shel
$ capture -d <path-to-video-device> -m -o -f \
 | gst-launch-1.0 -e fdsrc fd=0 ! rawvideoparse use-sink-caps=false format=yuy2 width=640 height=480 ! videoconvert ! xvimagesink sync=false
